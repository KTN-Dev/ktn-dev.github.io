Why using touch screens?

Before...

Workers wrote their activity on paper sheets.

Team leaders got back sheets and retranscribed them every day in Excel files.

And then some administrative people manually recorded the production in QAD.

That meant...

Very long data processing.

Possible loss of data.

Potential errors.

Multiple parallel systems.

Now:

Touch screens were placed in workshops.

Workers input their activity by themself.

Production activity is recorded in QAD without human intervention.

Production monitoring can be done in real time.

With touch screens:

Transactions are greatly simplified and fast.

The validity of data is checked in real time.

The web interface is user-friendly and configurable.

Workers are guided by the system during all the process.

Any standard USB barcode readers can be connected for a better experience.

Overview of the system

So, how does it work?

All data exchanges are made with SQL and HTTP queries.

Technologies

In user side:

- HTML 5

- CSS 3

- Javascript

In server side:

- Apache 2

- PHP 5

- MySql 5

On the production ground:

- Industrial touch devices

- 19" screen size

- Windows 7 Embedded

Conclusion

After 5 years...

Using touch screens brings a great deal of advantages compared to its implementation cost.

But it's not only time saving, efficiency or simplicity.

It's also a way to...

Involve everybody around a common project.

Raise awareness all the teams to the importance of the reliability of data recorded in the ERP.

And modernize the company.
